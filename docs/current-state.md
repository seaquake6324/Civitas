# 邑 1.0 当前状态

本文是当前已实现玩法、兼容基线和存档状态的权威摘要。公共模组版本固定为 `1.0`；内部开发暂不修改模组自身版本号。

## 不可破坏的基线

- 初始 `3×3` 腹地不会自动回退；领土保持四向连通，核心始终位于领土内。
- 扩张依赖建设、文礼、活动、边境状态和连续邻接，不依赖人口。
- 地表与地下文礼、活动分别保存；边境由领土拓扑派生，不保存第二份权威副本。
- 威胁、预警、三波入侵、荒废和安全回退不能删除玩家建筑。
- 世界修改只在服务端主线程执行；常规反馈不进入聊天栏。
- Xaero 兼容始终可选，缺少地图模组不能阻止核心模组加载。
- 旧城市、文明和威胁存档必须迁移；不能要求玩家删除世界。

## 居民与人口

- 居民拥有持久记录，远处保持虚拟状态，接近玩家时有界预热并实体化。
- 年龄分为儿童、少年、青年、壮年和老年；各阶段具有不同体型、生命、移动和攻击属性，青年以玩家属性为基准。
- 个体寿命稳定随机为 60–90 岁。寿终和正式入侵死亡均为永久死亡，不会复活。
- 性别只有男、女两种；正常确认后不可修改，管理员可用 `/cvt gender male|female <玩家或 NPC>` 修改。
- 种族包括人、猪人、牛人和羊人。当前没有种族数值差异；人有四套实体外观，其他种族各一套。
- 名字从东欧、西欧、北欧和南欧风格名字池稳定生成。
- “邑：居民”创造标签按种族分行，包含四族、五年龄段共 20 枚居民刷怪蛋。

## 婚姻、家庭与生育

- 支持玩家与玩家、玩家与 NPC、NPC 与 NPC 婚姻，并允许同性婚姻。
- 当前为单配偶制；青年、壮年、老年可婚，儿童和少年不可婚；不提供离婚。
- 配偶死亡后保留亡者家庭历史，幸存者可立即再婚。
- 生育不要求婚姻。儿童不可生育；少年、青年、壮年、老年的阶段修正为 `70% / 100% / 80% / 50%`。
- 只有异性组合可生育；同性家庭通过收养获得孩子。孩子始终是 NPC，种族稳定随机继承父母一方。
- 玩家孕期按服务器时间推进，离线后仍会到期生产。住房、食物、安全、文礼、活动、家庭和拥挤度只影响意愿与速度。
- 出轨状态可以记录，但当前没有社会后果。

## 孤儿、迁移、建筑与治安

- 孤儿只来自真实儿童失去父母或有明确来源的家庭脱离，不会凭空生成。
- 收养校验实际距离、有效住宅、家庭容量和意愿；含玩家监护人的家庭要求所有在世玩家监护人确认。
- 非出生人口来自有来源记录的流民家庭；已经定居的居民当前不会自主迁出。
- 建筑登记保存真实空间、设施、容量、入口和失效原因；扫描与页面均有硬上限。
- 仓储必须是已登记建筑内的真实容器，并由领主明确授权。当前不启用工作、生产或运输订单。
- 建城者默认掌握军事指挥权，可登记有上限的巡逻路线。
- 少年至老年居民可任守卫；强制任命仍保留原始意愿，武器、护甲、盾牌、班次和路线参与计算。
- 治安按区块保存可解释证据；渗透只来自已验证的未受控入口、城墙缺口、洞穴、地下黑暗区域或伪装入口。

## 存档与诊断

| 持久根 | schema |
|---|---:|
| 城市 | 5 |
| 文明 | 3 |
| 威胁与治安 | 9 |
| 人口与家庭 | 13 |
| 建筑 | 3 |
| 居民装备 | 1 |

- 所有持久根必须继续提供前向迁移、逐条损坏隔离和未来 schema 只读保护。
- 人口、建筑、治安、迁移、巡逻、实体化、后台战斗和网络页面具有扫描、队列、批次或实体上限。
- `/cvt debug` 显示原始输入、分项、修正、截断、队列、schema、迁移结果和耗时。
- 小地图文礼与活跃度显示数字等级 `1 / 5` 至 `5 / 5`；实际玩法仍使用连续原始值，五档只用于显示分类。

## 当前明确不做

- 地主、企业主、产业产权和成员军事授权。
- 工作、生产、岗位、订单和 NPC 真实物品运输。
- 满意度驱动的自主迁出。
- 饥饿、疾病和事故事件；未来若启用并致死，仍必须是永久死亡。
- 离婚、婚外关系后果、种族数值差异和年龄服装差异。

## 验证

- 快速测试：`.\gradlew.bat test`
- 完整构建：`.\gradlew.bat build`
- 产物：`build/libs/civitas-1.0.jar`
- 当前自动化基线为 252 项测试、0 失败。
- `run/` 包含本地测试客户端和测试世界，不清理、不迁移、不提交；不能用删除世界解决迁移问题。
