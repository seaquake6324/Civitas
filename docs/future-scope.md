# 邑后续范围

本文只记录已经确认但尚未实现的方向。任何会实质改变玩法的权限、公式和默认数值，在实现前仍须形成简短玩法契约。

## 权力与产业

- 玩家创建城池后，领主仍是最高治理权威。
- 地主拥有地图上的土地，企业主拥有地图上有真实地点的产业；同一主体可以兼任。
- 新建城池的创建者初始是地主，后续可考虑封建、授权和成员军事权转授。
- 玩家加入 NPC 城池、非建城身份及其参与方式属于后续方向，不得在缺少契约时自行实现。

## 工作、生产与物流硬约束

- 工作、生产和运输必须由订单驱动；NPC 不会无授权替玩家拿取物品。
- 玩家自己的企业由玩家在工作界面配置；NPC 企业可由其权利主体配置。
- 每份运输订单必须明确来源容器、目标容器、物品、槽位和数量。
- 来源和目标必须是地图上已登记建筑中的真实容器，仓储不得抽象化。
- NPC 必须在服务端主线程真实取出物品、放入自身携带槽、步行并真实交付；死亡时物品真实掉落。
- 不可见 NPC 不推进取货、物品位移或交付。禁止抽象库存、虚空物流和凭空生产。
- 软预留只能记录容器位置、槽位、物品、数量和订单修订。物品被取走、容器损坏或区块卸载时，订单暂停并解释原因，不复制也不吞物。

## 人性与迁出

- 在满意度和人性模型确认前，已经定居的居民不得自主迁出。
- 饥饿、疾病、事故和流民事件暂不启用；未来启用后造成的死亡仍为永久死亡。

## 实现前必须决定

- 领主、地主、企业主和普通成员分别能创建、修改、暂停哪些产业与订单。
- 土地产权与建筑或产业产权冲突时，谁拥有最终决定权。
- NPC 企业如何产生订单，以及玩家加入 NPC 城池后的参与方式。
- 满意度、人性和自主迁出模型的输入、反馈、成功条件、滥用边界与明确不做内容。
